# oh-my-workbench — UX 설계서

> 문서 버전: 0.2
> 작성일: 2026-04-21 (최종 수정: 2026-04-22)
> 참조: [PRD v0.4](./PRD.md), [ARCHITECTURE v0.2](./ARCHITECTURE.md)
>
> **표기 규약** — 각 섹션 말미 또는 항목에 ✅ 구현 / ⚠ 부분 / ❌ 미구현 / ⏳ v1.1 이상 을 명시한다. 설계는 그대로 보존하되 실제 상태를 함께 표기한다.

---

## 1. 디자인 원칙

1. **"한 눈에, 한 번의 클릭"** — 실행 모드에서 모든 주요 자원을 한 화면에 노출하고, 단일 클릭으로 실행된다. 모달·드릴다운은 실행 경로에서 사용하지 않는다.
2. **실행 모드 vs 편집 모드의 완전 분리** — 편집 UI(핸들·삭제 X 버튼·드롭 가이드 등)는 실행 모드에서 일절 노출되지 않는다. 모드 전환이 시각적으로 명확해야 한다.
3. **조용한 기본 상태** — 장식 최소화, 타일은 아이콘 + 레이블. 색상은 카테고리 식별용 악센트에 한정.
4. **키보드 우선 설계** — 모든 주요 조작이 단축키로 가능하도록 한다. 접근성 요건이자 업무 효율성의 핵심.
5. **편집은 안전하게** — 자동 저장 + Undo. 파괴적 조작(삭제)은 확인 다이얼로그. 드래그 중 "잘못 놓음"은 시각적으로 거부.

## 2. 화면 구조 (와이어프레임)

├──────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  ▌ 업무 시스템                                                          │  ← Section Header

**레이아웃 규칙**
- 최상단 Topbar 고정 (56px). 좌측 앱 타이틀, 중앙 검색 바, 우측 설정/편집 토글
- Section은 세로 스택. 섹션 헤더 좌측에 얇은 세로 막대(`▌`) 악센트 — 카테고리별 색상 배지와 다르게 섹션은 중립 회색
- Section 본문은 Category 카드의 CSS grid: `repeat(auto-fit, minmax(220px, 1fr))`
- Category 카드 내부: Item 타일의 grid: `repeat(auto-fit, minmax(80px, 1fr))` (중간 타일 크기 기준)

### 2.2 편집 모드 (같은 구조 + 편집 어포던스)

```
┌──────────────────────────────────────────────────────────────────────────┐
│  oh-my-workbench            [🔍 검색     ]  [설정]  [✓ 편집중]          │  ← 편집 배지
├──────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  ⋮▌ 업무 시스템                                    [+ 카테고리]  [🗑]   │  ← 섹션 도구
│  ┌──────────────────────┐ ┌──────────────────────┐                      │
│  │ ⋮ 사내 포털       🗑 │ │ ⋮ 이슈 트래커     🗑 │                      │
│  │ ┌──┐ ┌──┐ ┌──┐ [＋]│ │ ┌──┐ ┌──┐ [＋]      │                      │
│  │ │HR│ │경│ │총│     │ │ │JR│ │CN│           │                      │
│  │ │ x│ │ x│ │ x│     │ │ │ x│ │ x│           │                      │
│  │ └──┘ └──┘ └──┘     │ │ └──┘ └──┘           │                      │
│  └──────────────────────┘ └──────────────────────┘                      │
│                                                                          │
│                                                [+ 섹션 추가]             │
└──────────────────────────────────────────────────────────────────────────┘
```

**편집 모드에서 추가로 노출되는 어포던스**
- **드래그 핸들(`⋮`)**: 섹션/카테고리 좌측, 타일은 전체가 grabbable
- **삭제 버튼(`🗑` / 타일 상단 `x`)**: 우상단 8px 오프셋. 클릭 시 확인 다이얼로그
- **추가 버튼(`+`)**: 각 카테고리 타일 그리드 끝, 섹션 하단, 페이지 하단에 ghost 스타일
- **편집 배지**: Topbar 우측 토글 버튼이 "편집중" 상태임을 강조 (색상·텍스트 변화)

## 3. 모드 전환

### 3.1 편집 모드 진입 / 종료
- **진입** ✅: Topbar 편집 버튼 클릭 또는 `Ctrl+E`
  - 데일리 스냅샷은 편집 모드 진입이 아니라 **앱 구동 시 `config_load`** 시점에 자동 저장 (PRD §5.1 FR-6.1)
  - 편집 버튼이 앰버 색으로 강조되며, 섹션/카테고리 추가 버튼·삭제 X 버튼·내보내기/가져오기·단축키 편집 입력이 노출
- **종료** ⚠: Topbar 편집 버튼 재클릭 또는 `Ctrl+E`. 변경 사항은 실시간 자동 저장되므로 별도 토스트 없음. `Esc`로 종료는 ❌ 미구현.

### 3.2 모드별 동작 차이 (요약표, 구현 상태 반영)

| 조작 | 실행 모드 | 편집 모드 | 상태 |
|---|---|---|---|
| 타일 클릭 | **즉시 실행** | 편집 다이얼로그 열림 | ✅ |
| 타일 우측 X 버튼 | — | 삭제 (확인 다이얼로그 없이 즉시) | ⚠ 확인 다이얼로그 미구현 |
| 드래그 | 비활성 | 이동 | ✅ |
| 타일 더블클릭 | — | (설계상 편집 패널, 현재는 단일 클릭으로 대체) | ❌ |
| 타일 우클릭 | 컨텍스트 메뉴 | 컨텍스트 메뉴 | ❌ 미구현 |
| `Delete` 키 | — | 선택 항목 삭제 | ❌ 미구현 |
| `F2` | — | 레이블 인라인 편집 | ❌ 미구현 |

## 4. 드래그 앤 드롭 세부

### 4.1 드래그 가능한 계층
- 아이템 ↔ 아이템 (같은 카테고리 내 순서 변경)
- 아이템 → 다른 카테고리 (카테고리 간 이동)
- 카테고리 ↔ 카테고리 (같은 섹션 내/간)
- 섹션 ↔ 섹션 (페이지 내 순서)

### 4.2 드롭 타깃 시각화
| 상황 | 시각화 |
|---|---|
| 유효한 드롭 존에 hover | 존 전체 배경 `bg-accent-500/10` + 점선 외곽선 |
| 두 아이템 사이 삽입 지점 | 높이 2px 가로 바(또는 세로 바) accent 색상 |
| 잘못된 드롭 (예: 아이템을 Topbar 위로) | 커서에 🚫 오버레이, 원래 자리로 스냅백(150ms) |
| 드래그 중 원본 타일 | `opacity-40`로 고스팅 |
| 드래그 중 오버레이 | 원본 타일 복사본이 마우스를 따라옴 (dnd-kit DragOverlay) |

### 4.3 키보드 드래그 ❌ 미구현
현재는 `PointerSensor`만 등록되어 있어 마우스 드래그만 지원. `KeyboardSensor` 추가 및 상태바 힌트는 M3 범위.

### 4.4 대량 이동
- v1.0에서는 단일 선택만 지원. 다중 선택 DnD는 v1.x.

## 5. 아이템 편집 패널 (실제 구현: 모달 다이얼로그)

설계는 우측 슬라이드 드로어였으나, M2 구현에서는 중앙 모달 형태의 `AddItemDialog` 컴포넌트로 단순화되어 있다. 아래 레이아웃은 설계 기준이며 실제 다이얼로그는 필드 구성만 동일.

### 5.1 트리거 ✅
- 편집 모드에서 타일 **클릭**(현재는 더블클릭이 아니라 단일 클릭)
- 카테고리 끝의 `+` 타일 클릭으로 새 아이템 추가

### 5.2 패널 레이아웃 (설계 — 우측 슬라이드 드로어, 폭 420px)

```
┌────────────────── 아이템 편집 ─────────────────── × ┐
│                                                     │
│  타입       ○ 웹 URL    ● 프로그램                  │
│                                                     │
│  레이블     ┌────────────────────────────────┐     │
│             │ Visual Studio Code              │     │
│             └────────────────────────────────┘     │
│                                                     │
│  대상       ┌────────────────────────────────┐     │
│ (프로그램)  │ C:\...\Code.exe          [찾기]│     │
│             └────────────────────────────────┘     │
│                                                     │
│  실행 인자  ┌────────────────────────────────┐     │
│             │ --new-window                    │     │
│             └────────────────────────────────┘     │
│                                                     │
│  작업 폴더  ┌────────────────────────────────┐     │
│             │ %USERPROFILE%           [찾기] │     │
│             └────────────────────────────────┘     │
│                                                     │
│  권한       ○ 일반  ● 관리자 (UAC 프롬프트)          │
│             ⚠ 관리자 권한 실행 시 매번 UAC 프롬프트가 │
│                표시됩니다.                           │
│                                                     │
│  아이콘     ┌──┐  [자동 추출]  [검색]  [업로드]    │
│             │🖼│                                    │
│             └──┘                                    │
│                                                     │
│  태그       [#dev] [#daily]              [+ 태그]  │
│                                                     │
│  ─────────────────────────────────────────────────  │
│                        [복제]  [삭제]    [완료]    │
└─────────────────────────────────────────────────────┘
```

웹 URL 타입일 때는 "대상" 대신 `URL`·`브라우저`·`프로파일`·`시크릿(토글)` 필드 노출. "실행 인자/작업 폴더/권한"은 숨김.

### 5.3 저장 모델
- 다이얼로그 내 저장 버튼으로 명시적 커밋 → `configStore.addItem` / `updateItem` → 내부에서 `config_save` invoke.
- 스토어 반영은 동기적, 디스크 저장은 throttle 적용.
- "삭제"는 현재 확인 다이얼로그 없이 즉시 반영 (⚠ M3에서 확인 플로우 추가 예정).

## 6. 아이콘 선택 플로우

### 6.1 웹 URL 아이콘 (구현 현황)
M2에서는 **Simple Icons 검색만 구현** ✅. favicon 자동 조회 탭(❌)과 로컬 업로드 탭(❌)은 M3 이월.

- **Simple Icons 검색**: `AddItemDialog` 안에서 입력 필드 하단에 결과 그리드 표시. 영문 키워드 부분 일치, 다수 결과 노출. 선택 즉시 미리보기 + 아이템에 `icon: { kind: "simpleIcons", slug, color }` 저장.
- **아이템별 렌더 스타일** ✅ — `iconStyle: "auto" | "light" | "dark"` 필드로 배경 테마 대비 색 반전 제어. `auto`는 `useTheme`가 감지한 모드에 따라 자동 전환.

설계상 최종 목표 모달(M3):
```
┌──────────────── 아이콘 선택 ──────────────── × ┐
│ [Simple Icons] [URL favicon] [업로드]          │
│ ...
└────────────────────────────────────────────────┘
```

### 6.2 프로그램 아이콘 ✅
- 기본값: **자동 추출** (exe/lnk). `icon_extract_app` command가 base64 PNG를 반환하면 `ItemIcon`이 그대로 `<img src="data:image/png;base64,...">`로 표시.
- AUMID(UWP) 기반 추출은 ⏳ 미구현. 브랜드 앱의 Simple Icons 폴백 UI도 미구현 — 현재는 대상 경로에서 추출 실패하면 기본 placeholder 표시.

## 7. 검색 (FR-4)

### 7.1 진입 ✅
- Topbar 검색 바 포커스 또는 `Ctrl+K` (설계 초안의 `Ctrl+F`는 브라우저·OS 컨벤션 충돌 가능성을 피해 `Ctrl+K`로 변경됨)
- 실행 / 편집 모드 양쪽에서 활성

### 7.2 동작 ✅ (구현과 설계의 차이 반영)
- 입력 즉시 **아이템 레이블 + URL + 대상 경로**에 대해 대소문자 무시 부분 일치
- 구현: **검색 결과는 별도 리스트** (`SearchResults` 컴포넌트)로 교체 렌더링 — 기존 레이아웃의 페이드 방식은 채택하지 않음. 검색어가 비면 즉시 원래 그리드 복귀
- 결과 항목 클릭 → 즉시 실행. Enter/Shift+Enter 커스텀 동작은 ❌ 미구현
- `Esc`: 검색어 초기화 ✅

## 8. 전역 핫키 및 창 호출 (FR-11)

### 8.1 기본 단축키 (사용자 변경 가능) ✅
- `Ctrl+Alt+Space` — 메인 창 show/hide 토글 (Tauri `tauri-plugin-global-shortcut`)
- 편집 모드에서 Topbar의 "단축키" 입력 필드로 즉시 변경 가능. Enter 또는 포커스 아웃 시 `hotkey_reregister` command 호출.
- 창이 숨겨진 상태에서 단축키 → 창 표시 + 포커스 복원. 검색 바 자동 포커스는 ⏳ 미구현.
- 트레이 아이콘은 ❌ 미구현 (v1.x 범위).

### 8.2 충돌 처리 ✅
- `hotkey_reregister`가 실패하면 입력 필드에 붉은 테두리 + title 속성으로 에러 노출. 성공 시 기존 단축키를 해제하고 새 값으로 교체.

## 9. 키보드 단축키 맵

| 단축키 | 상황 | 동작 | 상태 |
|---|---|---|---|
| `Ctrl+Alt+Space` (기본) | 전역 | 메인 창 표시/숨김 (FR-11) | ✅ |
| `Ctrl+K` | 양 모드 | 검색 바 포커스 + 선택 | ✅ |
| `Ctrl+E` | 양 모드 | 편집 모드 토글 | ✅ |
| `Ctrl+Z` / `Ctrl+Y` / `Ctrl+Shift+Z` | 편집 모드 | Undo / Redo | ✅ |
| `Esc` | 검색 활성 | 검색 초기화 | ✅ |
| `Esc` | 편집 모드 | 편집 모드 종료 | ❌ |
| `Tab` / `Shift+Tab` | 양 모드 | 영역 간 포커스 이동 | ⚠ 브라우저 기본 동작만 |
| `↑↓←→` | 타일 그리드 포커스 | 인접 타일로 포커스 이동 | ❌ |

## 11. 설정 창

### 11.1 현재 구현 (M2) ⚠
**별도의 설정 창은 없음.** 설정은 Topbar에서 바로 노출·변경된다:

- **테마 사이클 버튼** (상시) — 시스템 → 다크 → 라이트 순 클릭 전환
- **전역 단축키** (편집 모드 전용) — Topbar 입력 필드에서 인라인 편집, Enter/blur 시 `hotkey_reregister`
- **내보내기 / 가져오기** (편집 모드 전용) — Topbar 버튼 → 파일 다운로드 / 업로드

### 11.2 M3 계획 — 별도 설정 창 도입 시
카테고리 (좌측 사이드바 탭 설계 유지):
1. **일반** — 테마(✅), 타일 크기 S/M/L(❌), 언어 한국어/영어
2. **단축키** — 전역 핫키(✅), 앱 내 단축키 표시/변경
3. **브라우저** — FR-8 구현 후. 기본 브라우저, 프로파일 목록
4. **가져오기/내보내기** — 현재 기능을 이쪽으로 이관, v1.1에서 북마크 가져오기 추가
5. **고급** — 시작 시 자동 실행(FR-12), 트레이 상주, 로그 폴더 열기, CDN 아이콘 조회 허용(D-1)
6. **정보** — 버전(`app_version` command 제공 ✅), 라이선스, PRD/아키텍처 문서 링크

## 12. 토스트 / 알림

- **위치**: 우하단 스택이 아니라 **헤더 바로 아래 고정 알림 바**(상단).

|---|---|---|

| 가져오기 실패 | "가져오기 실패: {err.details 또는 err.message}" | ✅ |

| 전역 핫키 충돌 | 입력 필드 테두리 붉게 + title 속성 툴팁 | ✅ (인라인 표시) |
| 편집 모드 종료 / 자동 저장 / 스냅샷 | 표시 안 함 | — |

- Hover(실행 모드): `bg-neutral-800` 배경 + 약한 scale(1.02, 80ms)
