# oh-my-workbench — UX 설계서

> 문서 버전: 0.1
> 작성일: 2026-04-21
> 참조: [PRD v0.3](./PRD.md), [ARCHITECTURE v0.1](./ARCHITECTURE.md)

---

## 1. 디자인 원칙

1. **"한 눈에, 한 번의 클릭"** — 실행 모드에서 모든 주요 자원을 한 화면에 노출하고, 단일 클릭으로 실행된다. 모달·드릴다운은 실행 경로에서 사용하지 않는다.
2. **실행 모드 vs 편집 모드의 완전 분리** — 편집 UI(핸들·삭제 X 버튼·드롭 가이드 등)는 실행 모드에서 일절 노출되지 않는다. 모드 전환이 시각적으로 명확해야 한다.
3. **조용한 기본 상태** — 장식 최소화, 타일은 아이콘 + 레이블. 색상은 카테고리 식별용 악센트에 한정.
4. **키보드 우선 설계** — 모든 주요 조작이 단축키로 가능하도록 한다. 접근성 요건이자 업무 효율성의 핵심.
5. **편집은 안전하게** — 자동 저장 + Undo. 파괴적 조작(삭제)은 확인 다이얼로그. 드래그 중 "잘못 놓음"은 시각적으로 거부.

**레이아웃 규칙**
- 최상단 Topbar 고정 (56px). 좌측 앱 타이틀, 중앙 검색 바, 우측 설정/편집 토글
- Section은 세로 스택. 섹션 헤더 좌측에 얇은 세로 막대(`▌`) 악센트 — 카테고리별 색상 배지와 다르게 섹션은 중립 회색
- Section 본문은 Category 카드의 CSS grid: `repeat(auto-fit, minmax(220px, 1fr))`
- Category 카드 내부: Item 타일의 grid: `repeat(auto-fit, minmax(80px, 1fr))` (중간 타일 크기 기준)

### 2.2 편집 모드 (같은 구조 + 편집 어포던스)

```
┌──────────────────────────────────────────────────────────────────────────┐
│  oh-my-workbench            [🔍 검색     ]  [설정]  [✓ 편집중]          │  ← 편집 배지
├──────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  ⋮▌ 업무 시스템                                    [+ 카테고리]  [🗑]   │  ← 섹션 도구
│  ┌──────────────────────┐ ┌──────────────────────┐                      │
│  │ ⋮ 사내 포털       🗑 │ │ ⋮ 이슈 트래커     🗑 │                      │
│  │ ┌──┐ ┌──┐ ┌──┐ [＋]│ │ ┌──┐ ┌──┐ [＋]      │                      │
│  │ │HR│ │경│ │총│     │ │ │JR│ │CN│           │                      │
│  │ │ x│ │ x│ │ x│     │ │ │ x│ │ x│           │                      │
│  │ └──┘ └──┘ └──┘     │ │ └──┘ └──┘           │                      │
│  └──────────────────────┘ └──────────────────────┘                      │
│                                                                          │
│                                                [+ 섹션 추가]             │
└──────────────────────────────────────────────────────────────────────────┘
```

**편집 모드에서 추가로 노출되는 어포던스**
- **드래그 핸들(`⋮`)**: 섹션/카테고리 좌측, 타일은 전체가 grabbable
- **삭제 버튼(`🗑` / 타일 상단 `x`)**: 우상단 8px 오프셋. 클릭 시 확인 다이얼로그
- **추가 버튼(`+`)**: 각 카테고리 타일 그리드 끝, 섹션 하단, 페이지 하단에 ghost 스타일
- **편집 배지**: Topbar 우측 토글 버튼이 "편집중" 상태임을 강조 (색상·텍스트 변화)

## 3. 모드 전환

### 3.1 편집 모드 진입 / 종료
- **진입**: Topbar 편집 버튼 클릭 또는 `Ctrl+E`
  - 진입 시점에 **데일리 스냅샷** 자동 저장(오늘 아직 저장 안 됐으면) — PRD D-3
  - 배경이 아주 미묘하게 톤 다운 (`bg-neutral-950` → `bg-neutral-900`), 편집 UI 어포던스 페이드 인 (150ms)
- **종료**: 편집 버튼 다시 클릭, `Esc`, 또는 `Ctrl+E` 재입력
  - 자동 저장이라 별도 "저장" 프롬프트 없음. 토스트 "편집 완료" 1.5s

### 3.2 모드별 동작 차이 (요약표)

| 조작 | 실행 모드 | 편집 모드 |
|---|---|---|
| 타일 클릭 | **즉시 실행** | 타일 선택 (outline 강조) |
| 타일 더블클릭 | — | 편집 패널(우측 드로어) 열림 |
| 타일 우클릭 | 컨텍스트 메뉴(열기/복사 URL/속성…) | 컨텍스트 메뉴(편집/복제/삭제) |
| 드래그 | 비활성 | 이동 |
| `Delete` 키 | — | 선택 항목 삭제 (확인) |
| `F2` | — | 레이블 인라인 편집 |

## 4. 드래그 앤 드롭 세부

### 4.1 드래그 가능한 계층
- 아이템 ↔ 아이템 (같은 카테고리 내 순서 변경)
- 아이템 → 다른 카테고리 (카테고리 간 이동)
- 카테고리 ↔ 카테고리 (같은 섹션 내/간)
- 섹션 ↔ 섹션 (페이지 내 순서)

### 4.2 드롭 타깃 시각화
| 상황 | 시각화 |
|---|---|
| 유효한 드롭 존에 hover | 존 전체 배경 `bg-accent-500/10` + 점선 외곽선 |
| 두 아이템 사이 삽입 지점 | 높이 2px 가로 바(또는 세로 바) accent 색상 |
| 잘못된 드롭 (예: 아이템을 Topbar 위로) | 커서에 🚫 오버레이, 원래 자리로 스냅백(150ms) |
| 드래그 중 원본 타일 | `opacity-40`로 고스팅 |
| 드래그 중 오버레이 | 원본 타일 복사본이 마우스를 따라옴 (dnd-kit DragOverlay) |

### 4.3 키보드 드래그
- `Space` / `Enter`: 선택 항목 픽업
- `↑↓←→`: 이동 방향 (다음/이전 삽입 지점으로)
- `Space` / `Enter`: 드롭
- `Esc`: 취소
- dnd-kit `KeyboardSensor` 내장 기능을 그대로 사용하되, 힌트는 상태바에 "Space로 선택, 방향키로 이동, Enter로 놓기" 표시

### 4.4 대량 이동
- v1.0에서는 단일 선택만 지원. 다중 선택 DnD는 v1.x.

## 5. 아이템 편집 패널

### 5.1 트리거
- 편집 모드에서 타일 **더블클릭**
- 컨텍스트 메뉴 → "편집"
- 새 타일 `+` 버튼 클릭

### 5.2 패널 레이아웃 (우측 슬라이드 드로어, 폭 420px)

```
┌────────────────── 아이템 편집 ─────────────────── × ┐
│                                                     │
│  타입       ○ 웹 URL    ● 프로그램                  │
│                                                     │
│  레이블     ┌────────────────────────────────┐     │
│             │ Visual Studio Code              │     │
│             └────────────────────────────────┘     │
│                                                     │
│  대상       ┌────────────────────────────────┐     │
│ (프로그램)  │ C:\...\Code.exe          [찾기]│     │
│             └────────────────────────────────┘     │
│                                                     │
│  실행 인자  ┌────────────────────────────────┐     │
│             │ --new-window                    │     │
│             └────────────────────────────────┘     │
│                                                     │
│  작업 폴더  ┌────────────────────────────────┐     │
│             │ %USERPROFILE%           [찾기] │     │
│             └────────────────────────────────┘     │
│                                                     │
│  권한       ○ 일반  ● 관리자 (UAC 프롬프트)          │
│             ⚠ 관리자 권한 실행 시 매번 UAC 프롬프트가 │
│                표시됩니다.                           │
│                                                     │
│  아이콘     ┌──┐  [자동 추출]  [검색]  [업로드]    │
│             │🖼│                                    │
│             └──┘                                    │
│                                                     │
│  태그       [#dev] [#daily]              [+ 태그]  │
│                                                     │
│  ─────────────────────────────────────────────────  │
│                        [복제]  [삭제]    [완료]    │
└─────────────────────────────────────────────────────┘
```

웹 URL 타입일 때는 "대상" 대신 `URL`·`브라우저`·`프로파일`·`시크릿(토글)` 필드 노출. "실행 인자/작업 폴더/권한"은 숨김.

### 5.3 저장 모델
- 모든 입력은 **디바운스 300ms + 즉시 store 반영**. store mutator가 자동으로 `config_save` 호출(200ms throttle).
- "완료" 버튼은 실제로는 **패널 닫기** 동작일 뿐(이미 저장됨). 학습 비용 완화를 위한 affordance.
- "삭제"는 확인 다이얼로그.

## 6. 아이콘 선택 플로우

### 6.1 웹 URL 아이콘
세 가지 소스를 하나의 모달로 통합:

```
┌──────────────── 아이콘 선택 ──────────────── × ┐
│                                                │
│ [Simple Icons] [URL favicon] [업로드]          │  ← 탭
│                                                │
│ ┌──────────────────────────────────────────┐  │
│ │ 🔍 github                                │  │  ← 영문 검색 (D-1)
│ └──────────────────────────────────────────┘  │
│                                                │
│  GitHub    GitLab   GitBook  Git      ...     │
│  ┌──┐      ┌──┐     ┌──┐    ┌──┐             │
│  │🐙│      │🦊│     │📗│    │ g│             │  ← 클릭하면 선택
│  └──┘      └──┘     └──┘    └──┘             │
│                                                │
│  색상      ● 브랜드(#181717)  ○ 흰색  ○ 검정  │  ← 렌더 옵션
│                                                │
│                          [취소]  [적용]        │
└────────────────────────────────────────────────┘
```

- **Simple Icons 탭**: 영문 키워드 부분 일치, 결과 상위 30개 가상 스크롤. 선택 즉시 하단 "렌더 옵션"에 브랜드 컬러·단색 표시.
- **URL favicon 탭**: 현재 입력된 URL의 favicon을 자동 조회, 결과 프리뷰. 실패 시 "이 사이트에서 favicon을 찾지 못했습니다" 안내.
- **업로드 탭**: 드래그 앤 드롭 또는 파일 선택. PNG/SVG/ICO 지원, 1MB 제한 안내. (D-1 폴백)

### 6.2 프로그램 아이콘
- 기본값: **자동 추출** (exe/lnk/UWP)
- 추출 결과 프리뷰. "다시 추출", "검색", "업로드" 버튼으로 대체 가능.
- Simple Icons 탭: 대상이 브랜드 앱(예: VS Code)이면 Simple Icons 검색이 유용 — 같은 모달 재사용.

## 7. 검색 (FR-4)

### 7.1 진입
- Topbar 검색 바 포커스 또는 `Ctrl+F`
- 실행 모드에서만 활성. 편집 모드 진입 시 검색은 자동 비활성(편집 중 텍스트 입력 방지).

### 7.2 동작
- 입력 즉시 **섹션/카테고리/아이템 레이블**에 대해 대소문자 무시 부분 일치
- 매칭되지 않는 타일은 `opacity-20`으로 페이드, 매칭된 타일은 원본 그대로
- 빈 카테고리/섹션도 페이드 처리
- Enter: 매칭 중 **첫 아이템 실행**, Shift+Enter: 새 창(브라우저 아이템 한정), Esc: 검색 초기화

## 8. 전역 핫키 및 창 호출 (FR-11)

### 8.1 기본 단축키 (사용자 변경 가능)
- `Ctrl+Alt+Space` — 메인 창 show/hide 토글
- 창이 숨겨진 상태에서 단축키 → 포커스 복원 + 검색 바 자동 포커스
- 트레이 아이콘 클릭 = 동일 동작

### 8.2 충돌 처리
- 설정에서 단축키 등록 실패 시 ("이미 사용 중") 인라인 에러 + 변경 유도

## 9. 키보드 단축키 맵

| 단축키 | 상황 | 동작 |
|---|---|---|

| `Tab` / `Shift+Tab` | 양 모드 | 영역(섹션/카테고리/타일 그룹) 간 포커스 이동 |
| `↑↓←→` | 타일 그리드 포커스 | 인접 타일로 포커스 이동 |
| `Enter` | 실행 모드 · 타일 포커스 | 아이템 실행 |

| `Space` / `Enter` | 편집 모드 · DnD 시작점 | 드래그 픽업 / 드롭 |
| `F2` | 편집 모드 | 인라인 레이블 편집 |
| `Delete` | 편집 모드 | 선택 항목 삭제 (확인) |
| `Ctrl+,` | 양 모드 | 설정 창 열기 |

## 10. 빈 상태 / 온보딩

### 10.1 최초 실행
- 환영 헤더 1줄: "시작을 위한 예시 구성을 만들어 두었습니다."
- 프리셋으로 다음 구성 seed:
  - Section "시작하기"
    - Category "웹"
      - Item: GitHub (https://github.com, Simple Icons `github`)
      - Item: Google (https://google.com)
    - Category "로컬"
      - Item: 메모장 (`notepad.exe`)
      - Item: 탐색기 (`explorer.exe`)
  - Section "내 업무"
    - Category "예시" — 빈 상태로, 가운데 "아이템 추가" 카드 1장만 있어 추가 진입점 제공

### 10.2 빈 카테고리
- "아직 아이템이 없어요. `+` 버튼 또는 편집 모드에서 추가하세요." 한 줄 힌트 + 점선 박스.

## 11. 설정 창

### 11.1 카테고리 (좌측 사이드바 탭)
1. **일반** — 테마(시스템/다크/라이트), 타일 크기(S/M/L), 언어(한국어/영어 — v1.0은 한국어 우선)
2. **단축키** — 전역 핫키, 앱 내 단축키 표시/변경
3. **브라우저** — 기본 브라우저, 브라우저별 프로파일 목록(감지된 것 표시)
4. **가져오기/내보내기** — 설정 JSON 내보내기/가져오기(FR-6), v1.1에서 북마크 가져오기 추가
5. **고급** — 자동 실행(시작 시), 트레이 상주, 로그 폴더 열기, CDN 아이콘 조회 허용(D-1, 기본 OFF)
6. **정보** — 버전, 라이선스, PRD/아키텍처 문서 링크

### 11.2 디자인 원칙
- 설정 창은 모달이 아닌 **독립 창** 또는 **우측 슬라이드 패널** — 본 화면을 계속 참조하며 설정 변경 가능
- 변경은 즉시 반영(확인 버튼 없음), 되돌리기는 "기본값으로 복원" 단일 버튼

| 설정 가져오기 성공 | "{count}개의 아이템을 가져왔습니다" | 성공 |
| 스냅샷 저장 | 표시 안 함 | — |

## 13. 시각 디자인 가이드

### 13.1 팔레트 (다크 테마 기본)
- 배경: `neutral-950` (본문) / `neutral-900` (편집 모드) / `neutral-800` (카드)

- 텍스트: `neutral-100` (기본) / `neutral-400` (보조) / `neutral-500` (힌트)

### 13.2 라이트 테마
- 배경: `white` / `neutral-50` / `neutral-100`
- 텍스트: `neutral-900` / `neutral-600` / `neutral-500`
- 악센트: 동일 계열을 `indigo-600`으로 톤 조정

### 13.3 타이포그래피
- 본문: system-ui stack (Windows에서는 Segoe UI)
- 코드/경로: `ui-monospace` (Cascadia Mono 우선)
- 크기: Topbar 14px, Section 18px bold, Category 13px medium, Tile label 12px

### 13.4 타일 크기(설정)
| 프리셋 | 아이콘 | 레이블 | 타일 총 폭 |
|---|---|---|---|
| S | 24px | 11px · 2줄 ellipsis | 64px |
| **M (기본)** | 32px | 12px · 2줄 | 80px |
| L | 48px | 13px · 2줄 | 104px |

### 13.5 상태 표시
- 포커스: 2px `indigo-500` outline + 4px offset
- Hover(실행 모드): `bg-neutral-800` 배경 + 약한 scale(1.02, 80ms)
- 선택(편집 모드): 2px `indigo-500` outline
- 관리자 실행 표시: 타일 우하단 작은 방패 배지(🛡)
